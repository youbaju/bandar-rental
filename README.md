# نظام إدارة إيجارات عمارة بندر

موقع لإدارة إيجارات العمارة: الشقق، المستأجرون، سجل الدفعات، مع تنبيهات تلقائية للعقود المتأخرة والقريبة من الانتهاء.

## الخطوات المتبقية قبل الاستخدام

### 1. ضبط قواعد الأمان في Firestore (مهم جداً)

بدون هذه الخطوة لن يعمل الموقع، لأن قاعدة البيانات حالياً تمنع أي قراءة أو كتابة.

1. افتح [Firebase Console](https://console.firebase.google.com) → مشروع `bandar-rental`
2. من القائمة الجانبية: **Databases & Storage → Firestore Database → Rules**
3. امسح المحتوى الموجود، والصق محتوى ملف `firestore.rules` المرفق هنا
4. اضغط **Publish**

### 2. رفع الملفات على GitHub Pages

ارفع كل ملفات هذا المجلد (`index.html`, `app.html`, `import.html`, مجلد `css`, مجلد `js`) إلى مستودع GitHub بنفس طريقتك المعتادة في مشاريعك الأخرى، وفعّل GitHub Pages من إعدادات المستودع.

### 3. مزامنة بيانات الأكسس (يمكن تكرارها بأمان)

بعد رفع الموقع:
1. افتح `index.html` وسجّل دخولك
2. بعد الدخول، غيّر الرابط يدوياً إلى `import.html` (مثال: `youbaju.github.io/bandar-rental/import.html`)
3. الصفحة تفحص قاعدة البيانات تلقائياً وتعرضلك عدد السجلات الجديدة قبل الاستيراد
4. اضغط **"ابدأ المزامنة"**

✅ الصفحة دي **ذكية** — بتقارن البيانات الموجودة في Firestore بالبيانات الجديدة، وما بتكررش أي سجل موجود بالفعل (بمقارنة رقم الشقة، رقم هوية المستأجر، وتاريخ العقد). يعني تقدر تشغّلها تاني بأمان كل ما بندر يبعتلك نسخة محدّثة من ملف الأكسس — يكفي تبعتلي الملف الجديد وأنا هحدّثلك البيانات المدمجة جوه `import.html`.

⚠️ **ملاحظة على تصحيح بيانات:** لاحظت إن سجل إيجار الشقة رقم 1 لشهر يونيو 2024 اتغير اسم المستأجر فيه من "عبدالله فهد الخليل" إلى "محمد علي مصلح الدعوس" في النسخة الجديدة من بندر. لو كنت عملت الاستيراد الأول قبل كده، هيتضاف السجل الصحيح الجديد **بجانب** السجل القديم الخطأ (مش هيستبدله تلقائياً) — راجع صفحة "سجل الإيجارات" في الموقع واحذف السجل القديم يدوياً لو لزم الأمر.

## الاستخدام

- **الرئيسية:** نظرة عامة على العمارة، مخطط مرئي للشقق ملوّن حسب الحالة، وتنبيهات فورية للعقود المتأخرة أو القريبة من الانتهاء (خلال 30 يوم)
- **الشقق:** إضافة/تعديل/حذف الوحدات وبياناتها
- **المستأجرون:** إدارة بيانات المستأجرين مع بحث سريع
- **سجل الإيجارات:** تسجيل دفعات جديدة، وعرض السجل الكامل مع بحث

## بنية البيانات في Firestore

- `flats`: الشقق (flatFloor, flatNumber, monthlRent, rented, meterNumber)
- `renters`: المستأجرون (renterId, renterName, nationality, mobile, workAddress)
- `rents`: سجلات الإيجار (renterId, renterName, flatNumber, dateOfRent, dateOfPay, amount, months, endOfRent, bank, paymentWay, reason)

## ملاحظات تقنية

- الحسابات: مستخدم واحد فقط (Firebase Authentication — Email/Password)
- لا توجد وظيفة طباعة إيصالات (تم إلغاؤها حسب الطلب)
- حالة كل شقة في المخطط الرئيسي تُحسب تلقائياً من آخر سجل إيجار مرتبط بها (وليست حقلاً مخزَّناً)
